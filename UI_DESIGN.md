## 一、设计原则

### 1.1 给什么样的人用

学术研究者：研究生、博士、博后、PI。熟悉 LaTeX、Python、命令行。愿意为"对的工具"投入学习成本，但讨厌臃肿和多余步骤。

### 1.2 设计主轴

| 原则 | 体现 |
|------|------|
| **agent 是主角，UI 是透视玻璃** | 不隐藏 agent 的操作，展示每个工具调用；不替代 agent 做决策 |
| **任何自动都可手动覆盖** | 表格、代码、样式、记忆 md 都能直接编辑 |
| **数据密集，不怕信息密度** | 学术用户习惯 overleaf/vscode，不需要 Apple 风格留白 |
| **单次交互低摩擦，长期使用可定制** | 新手看到快捷芯片就够用；老手掏出 memory.md 自定义 |
| **可追溯即可安心** | 永远能回到任何历史状态；破坏性操作必须可逆 |

### 1.3 视觉语言

- **底色**：暖米色 `#F5F1EA`（非纯白，降低长时间注视疲劳）
- **强调**：深墨 `#1C1917`（主要操作）、琥珀 `#B45309`（全局唯一最高优先操作 = 定稿）、青绿 `#0F766E`（成功/积极反馈）、紫罗兰 `#7C3AED`（agent / LLM 相关）
- **字体**：Fraunces（衬线斜体，标题）+ Geist（正文）+ JetBrains Mono（代码和数据）+ Cormorant Garamond（LaTeX 预览）
- **图表区**：永远白底，作为视觉锚点

## 二、整体布局

### 2.1 三种顶层视图

| 视图 | 用途 | 切入 |
|------|------|------|
| **Dashboard** | 项目总览 | 启动时默认；顶栏 "项目" |
| **Workspace** | 单个 task 的工作台 | 点击某 task 进入 |
| **Settings** | 全局设置 | 顶栏齿轮 |

顶部导航条在所有视图间共享，左侧 Logo + 面包屑，右侧视图切换 + 当前 LLM 徽章 + 设置。

### 2.2 Workspace 五列网格

```
┌──────┬──────────────┬──────────────┬──────────────┬──────┐
│ Proj │   Data       │   Preview    │   Context    │ Rail │
│ Nav  │              │              │   (tab body) │      │
│ 200  │  300-560     │   弹性       │   340        │  48  │
│ (收  │  (可扩展)     │              │   (可关闭)    │      │
│  缩)  │              │              │              │      │
└──────┴──────────────┴──────────────┴──────────────┴──────┘
```

**决策说明**：
- **最右侧的 Activity Rail（48px）永远可见**。它是 tab 切换器；对应的 tab 内容展开在 Context 栏。这比之前"tab 塞在 Context 头部"清爽得多
- **Data 栏可用全宽扩展**（560px）给大表格更多空间；原本"底部一半空白"的问题解决：表格 flex-1 占满，信息栏折叠
- **Context 栏整个可关闭**：全屏看图表时隐藏右两栏，图表区自动撑满

**新增特性**：
- **Tab 切换 raw / processed / pipeline**：表格上方有小 tab：`Raw (5)`、`Processed`、`Pipeline`

- **列类型徽章**：表头下一小行显示每列推断类型：`str / f64 / datetime`
- **选中高亮**：agent 讨论 "第 3 列" 时自动高亮，UI 反过来：用户选列后显示 "要对这列做什么？" 的 prompt

### 4.4 Preview（中栏）

和 v4 原型基本一致，几个增强：

**顶部工具栏**（从左到右）：
```
plot.py · 10:41 · ⭐        [git: 已保存]  [👁 编辑框] [↩][↪] [📊 对比]
```

**元素交互（FR-CHART-02）**：

```

│ 应用到                            │
│ [ 此柱 | 整组 | 全部 ]            │  ← 作用范围 toggle
├──────────────────────────────────┤
│ 颜色                   #0072B2   │
│ [调色板 swatches 8 色]            │
│ [🎨] [#0072B2] ←手输             │
├──────────────────────────────────┤
│ 透明度              ▁▂▃▄▅▆▇ 100% │
├──────────────────────────────────┤
│ 填充图案                          │
│ [∅] [/] [\] [|] [—] [+] [×] [·]  │
├──────────────────────────────────┤

└──────────────────────────────────┘
```

**"应用到" 三态**是关键：

- **全部**：所有系列都换（换主题）

点击外部关闭。修改立刻反映到图，松手 1.2s 后 git 提交。

### 4.6 Context Panel（右栏 340px · 五 tab）

右栏内容由 Activity Rail 控制切换。五个 tab：

| Icon | 名称 | 功能 |
|------|------|------|
| 💬 | **对话** | 与 agent 对话，看 agent 调用工具、回复 |
| 🕐 | **历史** | git 时间线，恢复 / 对比 / diff |
| 🎨 | **配色** | 学术配色方案库 |
| `</>` | **代码** | 查看 / 编辑 plot.py |
| 🧠 | **记忆** | 三级 md 的查看和编辑 |
| ⚙ | **模型** | LLM provider 配置 |

### 4.7 Activity Rail 设计

48px 宽的图标栏，紧贴右边缘。每个 tab 是一个图标按钮，当前激活的 tab：
- 图标变深色
- 左侧出现 2px 的深色竖条指示
- 背景变白

点击已激活的 tab → 整个右栏折叠（图表区获得更多空间）。再点击展开。

### 7.3 动画

- 模态淡入：200ms
- 面板切换：slide-in 300ms
- 按钮 hover：opacity 150ms
- 图表生成：骨架屏 + pulse 呼吸灯
- Git 状态徽章切换：颜色渐变 300ms

**避免**：弹跳、过度的 spring 动画、纸飞机飞出——对工程用户显得幼稚。

### 7.4 反馈规范

| 操作 | 反馈 |
|------|------|
| 按钮点击 | 背景变深 100ms |
| 保存成功 | 底部 toast "已保存 ✓"，2.4s 消失 |
| 破坏性操作 | 模态确认（不用 alert） |
| 长时任务 | 顶部 banner + 进度条，或 agent 区的 turn 动画 |
| 错误 | 红底 toast（避免吓人，不全屏） |
| 无操作几秒 | git 自动保存的无声提示（只有状态徽章变化） |
